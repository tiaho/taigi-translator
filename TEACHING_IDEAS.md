## Technical Considerations

### Data Storage Options
1. **localStorage:** Simple, client-side, survives page refresh
   - Good for: flashcards, progress tracking, preferences
   - Limits: ~5-10MB, no sync across devices

2. **Backend Database (PostgreSQL/MongoDB):** Server-side storage
   - Good for: user accounts, cross-device sync, leaderboards
   - Requires: authentication system, backend API endpoints

3. **Hybrid Approach:** localStorage + optional cloud sync
   - Start with localStorage only
   - Add backend sync as future feature

### Authentication
- **Not required initially:** Start with localStorage only
- **Future:** Add optional accounts for cloud sync
  - OAuth (Google, Facebook)
  - Email/password
  - Anonymous mode still available

### Mobile Considerations
- All features should work on mobile browsers
- Touch-friendly UI for drag-and-drop
- PWA (Progressive Web App) for installable app experience
- Offline mode for flashcard review
